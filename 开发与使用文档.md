# 模拟交易系统开发与使用文档

## 1. 项目概述

本系统根据《模拟交易程序需求规格说明书（简化版）》实现，采用 `Python + FastAPI + SQLAlchemy + MySQL` 构建，当前已形成一套可运行、可测试、可演示的模拟交易平台。

系统包含三类主要前端页面：

- 策略管理台
- 策略详情页
- 下单测试台

## 2. 本轮优化内容

### 2.1 年化收益率修正

原先年化收益率在样本周期过短时采用指数放大，容易出现极大值。现已改为：

- 当样本天数少于 30 天时，采用线性年化近似
- 当样本天数不少于 30 天时，采用复利年化
- 对展示值增加安全裁剪，避免异常极值破坏界面显示

### 2.2 收益曲线渲染修正

策略详情页收益曲线已改为更稳的 SVG 绘制方式：

- 增加面积填充
- 增加折线节点圆点
- 对单点和低波动场景做兼容显示

### 2.3 随机演示数据与接口验证

新增脚本 `seed_demo_data.py`，会：

- 随机生成演示策略
- 随机生成买入、卖出记录
- 更新现价
- 执行策略评估
- 验证主要页面和接口可用性

## 3. 技术实现

- 后端框架：FastAPI
- ORM：SQLAlchemy 2.x
- 数据库：MySQL 8.0+，测试兼容 SQLite
- 页面：Jinja2 + 原生 HTML/CSS/JavaScript
- 启动脚本：`run.py`
- 测试脚本：`run_tests.py`
- 演示数据脚本：`seed_demo_data.py`
- 本地依赖目录：`.vendor`

## 4. 已实现模块

### 4.1 策略管理

- 新增策略
- 分页查询策略列表
- 获取策略详情
- 修改策略内容
- 切换 `draft / active / archived`
- 归档策略只读
- 更新时版本号自动加 1
- 在策略列表中直接展示：
  - 累计收益
  - 年化收益
  - 今日收益
  - 最大回撤
  - 累计盈亏

### 4.2 策略详情

- 点击策略名称进入详情页
- 查看当前持仓
- 查看历史持仓变化
- 查看最近成交
- 查看收益曲线
- 查看策略级收益指标卡片

### 4.3 交易管理

- 激活策略才允许交易
- 支持买入和卖出
- 自动写入成交记录
- 卖出时校验持仓充足
- 自动计算已实现盈亏
- 支持历史成交分页查询
- 支持 CSV 导出
- 支持按策略过滤统计汇总

### 4.4 持仓管理

- 自动创建/更新持仓
- 维护加权平均成本
- 支持批量更新现价
- 自动计算持仓市值和浮动盈亏
- 支持全局和策略维度的组合总览接口
- 支持根据交易记录重建历史持仓变化

### 4.5 策略评估扩展字段

评估结果中新增以下字段：

- `strategy_return_pct`
- `strategy_annualized_return_pct`
- `benchmark_return_pct`
- `benchmark_annualized_return_pct`
- `excess_return_pct`
- `alpha`
- `beta`
- `sharpe_ratio`
- `sortino_ratio`
- `information_ratio`
- `win_rate`
- `profit_loss_ratio`
- `daily_win_rate`
- `profit_trade_count`
- `loss_trade_count`
- `strategy_volatility_pct`
- `benchmark_volatility_pct`
- `avg_daily_excess_return_pct`
- `excess_return_max_drawdown_pct`
- `excess_return_sharpe_ratio`
- `max_drawdown_period`

说明：基准收益当前基于 `benchmark_annual_return` 假设值生成，适用于模拟评估场景。

## 5. 运行方式

### 5.1 启动服务

```bash
python run.py
```

### 5.2 运行测试

```bash
python run_tests.py
```

### 5.3 生成演示数据并做接口烟雾验证

```bash
python seed_demo_data.py
```

### 5.4 数据库默认配置

```yaml
mysql:
  user: root
  password: 7842zc
  host: localhost
  database: trade_system_db
  port: 3306
```

程序会优先读取环境变量；未配置时使用以上默认值。

## 7. 已知限制

- 未接入真实外部基准行情，benchmark 仍基于假设年化收益率生成
- 历史持仓变化由交易流水重建，不是独立快照表
- 收益曲线仍缺少缩放、悬浮提示等交互
- 目前仍未实现独立的策略版本快照表
